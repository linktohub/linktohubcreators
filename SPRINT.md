## Task 1 — ✅ Fix Gelato shipping address + trigger order emails CRITICAL

**Why:** The checkout intent writes shipping to `pi.shipping` (Stripe's native shipping field), but the webhook at line 101 reads from `pi.metadata.shipping_address` — a field that is never set. `shippingAddress` is therefore always `null`, the `product.type === "merch" && shippingAddress && …` guard always fails, and **zero Gelato orders have ever fired**. Additionally, `sendOrderConfirmation` is never called anywhere — buyers receive no email after purchase.

**What to build:**

`src/app/api/checkout/webhook/route.ts` — change shipping extraction (lines 100–101) to fall back to the Stripe-level `pi.shipping` object:

```ts
const shippingName  = pi.metadata.shipping_name  || pi.shipping?.name || "";
const shippingAddress = pi.metadata.shipping_address
  ? JSON.parse(pi.metadata.shipping_address)
  : pi.shipping?.address
  ? {
      firstName:   (pi.shipping.name || "").split(" ")[0] || "",
      lastName:    (pi.shipping.name || "").split(" ").slice(1).join(" ") || "",
      addressLine1: pi.shipping.address.line1   || "",
      city:         pi.shipping.address.city    || "",
      postCode:     pi.shipping.address.postal_code || "",
      country:      pi.shipping.address.country || "US",
    }
  : null;
```

Also add a `sendOrderConfirmation` call after the `downloadUrls` array is built (before the `increment_creator_revenue` RPC). Pass items as `{ name: p.title, quantity: item.quantity, unitPrice: Math.round(p.price * 100) }`.

`src/lib/email.ts` — change the `getResend()` guard from `throw` to a graceful return so a missing key doesn't crash the webhook mid-order:

```ts
function getResend(): Resend | null {
  const key = process.env.RESEND_API_KEY;
  if (!key || key === "your_resend_api_key") {
    console.warn("[email] RESEND_API_KEY not configured — skipping email");
    return null;
  }
  return new Resend(key);
}
```
Update `sendOrderConfirmation` and `sendNewSubscriberNotification` to early-return when `getResend()` returns null.

Also fix Gelato `files` field: pass `images[0]` (the creator-uploaded product image) as the design print file so orders don't submit blank:

```ts
files: product.images?.[0]
  ? [{ type: "default", url: product.images[0] }]
  : [],
```

**Done when:**
- A test merch purchase with a shipping address produces a Gelato order attempt (visible in server logs or Gelato dashboard)
- A test purchase triggers an order confirmation email when `RESEND_API_KEY` is set; if not set, webhook completes without crashing

**Deployment note:** Set `RESEND_API_KEY` in Vercel → Project Settings → Environment Variables. Verify the `linktohub.com` sending domain in the Resend dashboard.
