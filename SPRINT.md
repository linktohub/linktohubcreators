## Task 1 — Email Drip Sequences (D3 + D7) CRITICAL ✅ DONE

**Why:** The welcome email shipped in the 2026-06-22 sprint — the prerequisite is live. D3 and D7 drip sequences are now ready to build with zero new prerequisites. Stan charges $99/month for email automation; Beacons limits free tiers to 50 sends/month. Linktohub at base price with working drip sequences is the answer to the single most-cited creator complaint in 2026. Every creator who activates and gets D3/D7 sends is more likely to stay and drive sales from their storefront. RESEARCH confirms this is the highest-impact unbuilt feature on the board.

**What to build:**

**Step 1 — Migration** `supabase/migrations/20260720000001_email_drips.sql`:
```sql
ALTER TABLE public.email_subscribers
  ADD COLUMN IF NOT EXISTS drip_d3_sent_at timestamptz,
  ADD COLUMN IF NOT EXISTS drip_d7_sent_at timestamptz;

CREATE INDEX IF NOT EXISTS idx_email_subscribers_drip_d3
  ON public.email_subscribers (creator_id, drip_d3_sent_at)
  WHERE subscribed = true AND drip_d3_sent_at IS NULL;

CREATE INDEX IF NOT EXISTS idx_email_subscribers_drip_d7
  ON public.email_subscribers (creator_id, drip_d7_sent_at)
  WHERE subscribed = true AND drip_d7_sent_at IS NULL;
```

**Step 2 — Email functions** in `src/lib/email.ts`, add two functions after `sendSubscriberWelcome`:

`sendDripD3({ to, subscriberName, creatorName, creatorUsername, topProducts })`:
- Subject: `"3 days in — here's what ${creatorName} has for you"`
- Body: "Hey {name}! You joined {creator}'s list 3 days ago. Here's what's available in the store:" + top 3 products with prices + "Visit the store →" CTA linking to `/${creatorUsername}?utm_source=email&utm_medium=drip&utm_campaign=d3`

`sendDripD7({ to, subscriberName, creatorName, creatorUsername })`:
- Subject: `"Your first week with ${creatorName} — share this"`
- Body: "You've been part of {creator}'s community for a week. Know someone who'd love their content? Share the store:" + share link with UTM + "Visit the store →" CTA

**Step 3 — Drip API route** `src/app/api/email/drip/route.ts`:
```ts
// POST — called by Vercel Cron. Authenticate with CRON_SECRET bearer token.
// D3 batch: query email_subscribers WHERE subscribed = true
//   AND created_at <= now() - interval '3 days'
//   AND drip_d3_sent_at IS NULL
//   LIMIT 100 (Resend rate safety)
// For each subscriber: fetch creator and their top 3 products by price desc,
//   call sendDripD3(), update drip_d3_sent_at = now()
// D7 batch: same pattern with drip_d7_sent_at
// Return { d3_sent: N, d7_sent: N }
```
Use `createAdminClient()` (service role) to bypass RLS — cron has no user session.
Add `CRON_SECRET` to Vercel env vars and check `Authorization: Bearer ${CRON_SECRET}` in the route.

**Step 4 — Vercel Cron** in `vercel.json`, add:
```json
"crons": [
  {
    "path": "/api/email/drip",
    "schedule": "0 10 * * *"
  }
]
```
(Runs daily at 10:00 UTC. Vercel Cron sends a `POST` with `Authorization: Bearer {CRON_SECRET}`.)

**Done when:** A test subscriber created 3+ days ago receives the D3 email. A subscriber created 7+ days ago receives the D7 email. `drip_d3_sent_at` / `drip_d7_sent_at` are set so the email never fires twice. The Vercel Cron job appears in the dashboard at linktohub.vercel.app's Vercel project settings. No email fires to unsubscribed (`subscribed = false`) rows.

**Env vars to add in Vercel:** `CRON_SECRET` (any long random string)

## Task 5 — Subscription Tier "Most Popular" Badge MEDIUM ✅ DONE

**Why:** DESIGN_NOTES (2026-07-15 pass) flags this as medium priority: "Subscription tier cards lack visual differentiation — a 'Most popular' badge on the middle tier would drive more conversions." Currently all subscription tiers on storefronts render with identical dark cards and no visual hierarchy. When a creator has 2–3 tiers, buyers default to the cheapest or stall. A visual highlight on the middle tier reliably shifts buyers toward a higher-value plan — this is standard conversion practice on every SaaS pricing page and directly increases creator subscription revenue.

**What to build:**

In `src/app/[username]/storefront-client.tsx`, the `tiers.map()` render loop (around line 855):

```tsx
{tiers.map((tier, idx) => {
  const isPopular = tiers.length > 1 && idx === Math.floor(tiers.length / 2);
  return (
    <div key={tier.id} className={cn(
      "rounded-2xl p-6 transition-all",
      isPopular
        ? "border border-violet-500/40 bg-violet-500/[0.05] ring-1 ring-violet-500/20"
        : "border border-white/[0.08] bg-white/[0.02]"
    )}>
      {isPopular && (
        <span className="inline-block text-[10px] font-black tracking-widest uppercase
                         text-violet-300 bg-violet-500/20 px-3 py-1 rounded-full mb-3">
          Most popular
        </span>
      )}
      {/* existing tier content unchanged */}
```

Logic: `Math.floor(tiers.length / 2)` picks index 0 for 1 tier (never shown — badge requires `tiers.length > 1`), index 1 for 2–3 tiers, index 2 for 4 tiers. Middle tier gets the highlight.

The Subscribe button for the popular tier: bump from existing style to use `btn-gradient` class (matching the primary CTA style elsewhere on the storefront) instead of the current bordered button, reinforcing it as the recommended action.

**Done when:** A storefront with 2+ subscription tiers shows a violet "Most popular" badge on the middle tier, with a subtle violet ring and background tint distinguishing it from the other cards.
