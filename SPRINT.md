## Task 1 — Auto-generate PDF at purchase when file_url is missing CRITICAL

**Why:** A buyer pays for a digital product → webhook creates a `purchase_token` → confirmation email arrives with download link → `/api/download?token=X` looks up `products.file_url` → if null, returns `"File not found"` (HTTP 404). This happens whenever a creator creates a digital product without clicking "Generate PDF" in their dashboard first. Money in, broken delivery out. Silent failure, no buyer recourse.

**What to build:**

Extract the core PDF generation logic from `src/app/api/generate-pdf/route.ts` into a shared function in `src/lib/pdf-generator.ts`:

```ts
export async function generateAndStorePdf(productId: string, admin: SupabaseClient): Promise<string | null>
```

This function takes a `productId`, fetches the product + creator from DB, generates the PDF with Claude (claude-haiku), uploads to Supabase Storage `products/digital/{productId}/content.pdf`, updates `products.file_url`, and returns the public URL. Returns `null` on failure.

In `src/app/api/checkout/webhook/route.ts`, inside the `if (product.type === "digital" && customerEmail)` block — before creating the purchase token:

```ts
let fileUrl = product.file_url;
if (!fileUrl && (product.file_type === "pdf" || product.file_type === "course")) {
  try {
    fileUrl = await generateAndStorePdf(product.id, admin);
  } catch (err) {
    console.error("[webhook] auto-generate PDF failed:", err);
    // continue without file_url — token created, buyer gets "file generating" message
  }
}

const { data: token } = await admin.from("purchase_tokens").insert({
  order_id: order.id,
  product_id: product.id,
  buyer_email: customerEmail,
  delivery_pending: !fileUrl,   // new column — flags to dashboard if delivery incomplete
}).select("token").single();
```

Update `src/app/api/generate-pdf/route.ts` to import and delegate to `src/lib/pdf-generator.ts` (avoid code duplication).

Add `delivery_pending boolean default false` column to `purchase_tokens` table via Supabase migration.

**Done when:** A buyer who purchases any digital product — including one where the creator never ran "Generate PDF" — receives a working download link. `/api/download` returns a redirect to a real PDF, not a 404.

## Task 2 — Fix welcome email (storefront promise broken on every signup) CRITICAL

**Why:** `handleEmailCapture()` in `src/app/[username]/storefront-client.tsx` shows this success state after a fan subscribes:
> "You're in! ✓ · Check your inbox for a welcome from [creator]"

No welcome email is sent. `/api/notify/new-subscriber` only emails the *creator* ("You have a new subscriber"). Every fan who subscribes to any storefront email list is being told to expect an email that never arrives. This damages creator credibility and subscriber trust on the first interaction.

**What to build:**

Add to `src/lib/email.ts`:

```ts
export async function sendSubscriberWelcome({
  to, subscriberName, creatorName, creatorUsername, topProducts,
}: {
  to: string;
  subscriberName?: string;
  creatorName: string;
  creatorUsername: string;
  topProducts: { title: string; price: number; type: string }[];
}) {
  const resend = getResend();
  if (!resend) return;
  const storefrontUrl = `https://linktohub.vercel.app/${creatorUsername}`;
  const productList = topProducts.slice(0, 3).map(p =>
    `<li style="margin:6px 0"><a href="${storefrontUrl}" style="color:#7c3aed">${p.title}</a> — $${p.price}</li>`
  ).join("");

  await resend.emails.send({
    from: `${creatorName} <updates@linktohub.com>`,
    to,
    subject: `Welcome to ${creatorName}'s inner circle!`,
    html: `<div style="font-family:sans-serif;max-width:520px;margin:0 auto;color:#111">
      <h2 style="color:#7c3aed">You're in! 🎉</h2>
      <p>Hey${subscriberName ? ` ${subscriberName}` : ""}! Thanks for joining <strong>${creatorName}</strong>'s community.</p>
      ${productList.length ? `<p><strong>Check out what's in the store:</strong></p><ul>${productList}</ul>` : ""}
      <p><a href="${storefrontUrl}" style="color:#7c3aed;font-weight:bold">Visit the store →</a></p>
      <hr style="margin:24px 0;border:none;border-top:1px solid #eee">
      <p style="color:#888;font-size:12px">You subscribed at ${creatorName}'s Linktohub storefront. <a href="${storefrontUrl}/unsubscribe?email=${encodeURIComponent(to)}" style="color:#888">Unsubscribe</a></p>
    </div>`,
  });
}
```

Update `src/app/api/notify/new-subscriber/route.ts` to:
1. Fetch the creator's top 3 products from DB (ordered by `price desc`, `active = true`)
2. Call `sendSubscriberWelcome()` to the fan alongside the existing creator notification

**Done when:** A fan who enters their email on any storefront receives a welcome email from the creator within 60 seconds. The email includes the creator's name, a link to the storefront, and up to 3 product highlights.
